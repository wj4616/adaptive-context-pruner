# adaptive-context-pruner

Compresses long-running Claude Code conversation histories into structured domain-adaptive context documents via relevance-graph scoring and tier-aware eviction.

**v2.0.0** extends to dual-mode operation:
- **MANUAL** — explicit invocation, full pipeline
- **AUTOMATIC** — background micro-cycle triggered by token-utilization thresholds and scheduled intervals

With a hot/warm/cold tier architecture, dependency-graph critical-path protection, semantic episode clustering, and delta/diff file caching.

## Architecture

- **20 nodes / 39 edges / 9 waves** — full GoT + wave-modular + conditional-path topology
- **Determinism:** non-deterministic
- **Spawn budget:** ≤3 STANDARD/DEEP, ≤2 MINIMAL
- **Relevance formula:** `R(msg) = 0.3×recency + 0.3×semantic_similarity + 0.2×graph_centrality + 0.2×user_importance`
- **Domain set:** `{technical-debugging, creative-brainstorming, auto-detect, research, planning}`
- **Auto-trigger thresholds:** Yellow ≥0.70 (advisory) / Orange ≥0.85 (proactive prune) / Red ≥0.95 (emergency prune); scheduled every 15 turns

## Invocation

```
# MANUAL mode — explicit full pipeline
/adaptive-context-pruner \
  --input '{"conversation_history": [...], "session_metadata": {"turn_count": N, "total_tokens_estimate": T, "domain_hint": "technical-debugging", "recent_window_size": 20, "explicit_retention_directives": [...], "mode": "manual"}}'

# MANUAL --mode MINIMAL  — ≤15 min, 2 spawns, no clustering/caching
# MANUAL --mode DEEP     — highest fidelity, up to 2 recovery cycles

# AUTOMATIC mode — background micro-cycle (mode auto-detected from input shape)
/adaptive-context-pruner \
  --input '{"turn_delta": {...}, "current_token_count": T, "current_graph_state": {...}, "pending_tool_calls": [...]}'
```

## Hard gates (preserved from v1; upgrades noted)

- `[GRAPH-REPR]` Conversation as graph (turn = node, relevance = edge)
- `[DYNAMIC-SCORING]` Explicit dynamic relevance, not a fixed sliding window — *v2 upgraded formula*
- `[PRESERVE-INSTRUCTIONS]` Critical instructions always preserved
- `[PRESERVE-RECENT]` Recent context (configurable window) always preserved verbatim
- `[COMPRESS-NOT-DROP]` Older less-relevant turns compressed, not discarded
- `[EDGE-TOPIC-SWITCH]` Don't prune active topic thread on switch
- `[EDGE-CORRECTIONS]` Repeated corrections: latest overrides
- `[EDGE-MIND-CHANGE]` Mark superseded; retain in `## Superseded Instructions`; emit `conflict_signal` on unresolvable contradiction
- `[DOMAIN-ADAPT]` Strategy adapts per domain — *v2 domain set extended*
- `[OUTPUT-STRUCTURED]` Output consumable by any downstream agent
- `[NO-EXTERNAL-TOOLS]` Internal token counting only
- `[SELF-CONTAINED]` Standard Claude Code capabilities only

See `SKILL.md` for the full hard-gate inventory and contract.
