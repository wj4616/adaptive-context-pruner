## Â§5 Optimizations

1. **HC-23 dual-wave parallel dispatch (W2 4-node + W3 5-node):** All inline analysis nodes execute in a single response per wave, maximizing parallelism without spawns. W2 dispatches N-GENERATOR, N-CLASSIFIER, N-FILTER, N-FILE-CACHE as one unit; W3 dispatches N-SCORER-TECHNICAL, N-SCORER-CREATIVE, N-SEMANTIC-CLUSTER, N-ANALYZER-TOPIC-SWITCH, N-ANALYZER-CORRECTIONS as one unit. Estimated wall-clock reduction vs sequential: 4-5x per wave.

2. **MINIMAL mode static-spawn reduction (3â†’2):** N-TIER-MANAGER downgrades to inline model-medium in MINIMAL, saving one spawn allocation and reducing latency target from â‰¤25 min to â‰¤15 min. Three additional nodes become inactive (N-FILE-CACHE, N-SEMANTIC-CLUSTER, N-CONTINUITY-MARKER); back-edge caps set to 0 (E-20/E-21 never fire). Token ceiling drops from ~63,500 to ~55,500.

3. **Delta file caching (N-FILE-CACHE W2 â†’ N-REFINER W5):** SHA-256 comparison reduces file content tokens from full-text to ~15-token FileRef for unchanged files. AP-V35 / SC-12 guard ensures FileRef stays <20 tokens. Long-carry edge E-15c carries file_cache_map from N-FILE-CACHE (W2) to N-REFINER (W5), spanning 3 waves. Highest per-run token savings for sessions with large attached file context.

4. **N-TRIGGER-EVAL early-exit (E-02b NO-OP terminal):** Auto-mode below threshold exits after W1 (3 nodes only), eliminating full pipeline execution for the common no-prune case. Cost of a no-op run: ~4,500 tokens vs ~63,500 tokens for a full STANDARD run.

5. **5-tier priority merge in N-AGGREGATION vs 3-tier in v1:** Critical-path protection at tier 1 ensures pending tool-call context is never evicted regardless of R(msg) scores. The null-tolerant design for 3 of 8 streams (technical_scores, creative_scores, cluster_summaries) prevents AND-join stalls on domain-agnostic or MINIMAL runs.

6. **W7-pass-2 recovery path:** N-RECOVERY â†’ E-21 back-edge â†’ N-REFINER second pass â†’ N-VERIFIER second pass â†’ E-19b â†’ N-TIER-MANAGER allows corrective re-eviction without full pipeline restart. Back-edge caps encode mode policy: STANDARD=1 (at most 1 recovery cycle), MINIMAL=0 (no recovery â€” fast exit), DEEP=2 (up to 2 recovery cycles for highest fidelity). Recovery path adds at most N-REFINER(8,000) + N-VERIFIER(4,000) = 12,000 tokens per cycle.

7. **N-CRITICAL-PATH merged into N-GENERATOR (FD-02):** The brief-proposed standalone N-CRITICAL-PATH node was absorbed into N-GENERATOR's output ports (dependency_graph with critical=True flags on ContextNodes). This preserves the 20-node budget, eliminates an extra W2 fan-out slot, and keeps critical-path computation co-located with the dependency DAG construction â€” reducing the chance of AP-V33 shallow-ancestor computation errors.

8. **N-CONTINUITY-MARKER kept standalone (FD-14):** Rather than inlining continuity-marker generation into N-FORMATTER, N-CONTINUITY-MARKER remains a declared graph node. This preserves E-23 as a graph-declared forward-conditional edge per AP-V29, making the AP-V37 guard (MUST NOT fire when auto_prune_triggered==false) structurally enforceable rather than an invisible runtime check. The token overhead (~1,500) is justified by runtime-state visibility.

9. **R(msg) formula explicit and shared across scorer nodes (FD-18):** v1 used ad-hoc per-domain weights; v2 applies the canonical formula R(msg) = 0.3Ă—recency + 0.3Ă—semantic_similarity + 0.2Ă—graph_centrality + 0.2Ă—user_importance in both N-SCORER-TECHNICAL and N-SCORER-CREATIVE. This satisfies SC-10 (AP-D3 threshold declaration) and enables deterministic comparison across domains for N-AGGREGATION's domain-blend tier-5 merge.

## Â§6 GoT Controller

The orchestrator dispatches nodes by wave. Inline nodes within a wave execute in the same response (HC-23 parallel); spawn nodes require a separate agent call.

```python
# adaptive-context-pruner v2.0.0 â€” GoT Controller
def run_pipeline(input_record, mode_config):
    # W1: Sequential gate trio
    attention_reset_read()
    ingest_record = N_INGEST(input_record)
    preflight = N_PREFLIGHT(ingest_record, mode_config)
    if not preflight.pass_:
        return REFUSE_OUTPUT(preflight.refuse_signal)
    trigger = N_TRIGGER_EVAL(preflight.session_meta, mode_config)
    if trigger.mode == 'auto' and not trigger.trigger_signal:
        return NO_OP_TERMINAL()  # E-02b: below threshold, no pruning needed

    # W2: 4-way parallel fan-out (HC-23 single-response)
    # N-FILE-CACHE depends on N-FILTER.avoid_cache_hint within W2 â€” sequential sub-dependency
    [generator_out, classifier_out, filter_out, file_cache_out] = parallel([
        N_GENERATOR(trigger.session_meta),
        N_CLASSIFIER(trigger.session_meta),
        N_FILTER(trigger.session_meta),
        N_FILE_CACHE(trigger.session_meta, filter_out.avoid_cache_hint)
            if mode_config.enable_delta_cache else None
    ])

    # W3: 5-way parallel fan-out (HC-23 single-response)
    [tech_scores, creative_scores, cluster_summaries, topic_signal, supersession_map] = parallel([
        N_SCORER_TECHNICAL(generator_out.graph_structure)
            if classifier_out.domain_gate_technical else null_signal(),    # E-09 gate
        N_SCORER_CREATIVE(generator_out.graph_structure)
            if classifier_out.domain_gate_creative else null_signal(),     # E-10 gate
        N_SEMANTIC_CLUSTER(generator_out.dependency_graph, topic_out.episode_boundaries)
            if mode_config.enable_semantic_clustering and mode != 'MINIMAL' else null_signal(),
        N_ANALYZER_TOPIC_SWITCH(generator_out.graph_structure),
        N_ANALYZER_CORRECTIONS(generator_out.graph_structure)
    ])

    # W4: N-AGGREGATION spawn (AND-join; 8 streams; 3 null-tolerant)
    attention_reset_read()
    pruning_plan = N_AGGREGATION(
        tech_scores,                         # E-11 null-tolerant
        creative_scores,                     # E-12 null-tolerant
        cluster_summaries,                   # E-14b null-tolerant
        topic_signal,                        # E-13 required
        supersession_map,                    # E-14 required
        filter_out.protected_node_set,       # E-15 required
        ingest_record,                       # E-05b long-carry required
        generator_out.dependency_graph       # E-08d required
    )

    # W5/W6/W7: Refinement-verification-recovery loop (back-edge E-20/E-21)
    retry_count = 0
    recovery_overrides = None
    while True:
        attention_reset_read()
        # W5: N-REFINER spawn
        refined_context = N_REFINER(
            pruning_plan,                                               # E-16 required
            ingest_record,                                              # E-16b long-carry required
            file_cache_out.file_cache_map if file_cache_out else None,  # E-15c optional
            recovery_overrides if retry_count > 0 else None             # E-21 back-edge conditional
        )

        # W6: N-VERIFIER (inline, 13-SC battery)
        verify_result = N_VERIFIER(refined_context)  # E-18

        if verify_result.verify_pass:
            break  # E-19: proceed to N-TIER-MANAGER

        # E-20 back-edge fires: N-RECOVERY
        back_edge_cap = {'MINIMAL': 0, 'STANDARD': 1, 'DEEP': 2}[mode_config.mode]
        if retry_count >= back_edge_cap:
            break  # cap exceeded; proceed degraded with recovery_override_final=True
        recovery_overrides = N_RECOVERY(verify_result.violation_signal)  # E-20 -> E-21
        retry_count += 1
        # E-21 back-edge: re-enter W5 loop

    # W7: N-TIER-MANAGER (spawn STANDARD/DEEP; inline model-medium MINIMAL)
    # Fires via E-19 (verify_pass path) or E-19b (recovery_complete path)
    attention_reset_read()
    tier_result = N_TIER_MANAGER(
        verify_result.verify_pass_signal if verify_result.verify_pass else None,  # E-19
        recovery_overrides if retry_count > 0 else None,                          # E-19b path
        filter_out.protected_node_set,       # E-17 long-carry
        generator_out.dependency_graph       # E-08d long-carry
    )
    # tier_result emits tier_state (E-17b) + tier_state_final (E-22)

    # W8: N-FORMATTER + N-CONTINUITY-MARKER (conditional E-23)
    formatted = N_FORMATTER(
        filter_out.protected_node_set,  # E-17 long-carry
        tier_result.tier_state,         # E-17b
        tier_result.tier_state_final    # E-22
    )
    if formatted.auto_prune_triggered:  # E-23 forward-conditional
        continuity = N_CONTINUITY_MARKER(formatted.formatted_output)  # E-23 -> E-24
        final_output = continuity.continuity_output
    else:
        final_output = formatted.rendered_output  # E-24b direct to N-PERSISTER

    # W9: N-PERSISTER (no-llm, terminal)
    return N_PERSISTER(final_output)  # E-25 -> SKILL_OUTPUT
```

## Â§7 Pipeline Narrative

The adaptive-context-pruner v2.0.0 pipeline opens with a three-node sequential gate trio in Wave 1. N-INGEST normalizes the incoming request into a structured ingest record regardless of whether the skill is invoked in `manual` mode (full conversation history supplied) or `auto` mode (incremental turn delta with current token count and graph state). N-PREFLIGHT performs fail-fast structural validation â€” malformed input or missing required fields routes immediately to REFUSE_OUTPUT via the E-02 terminal edge, preventing any wasted downstream computation. N-TRIGGER-EVAL evaluates the dual-mode entry path: in auto mode, it checks token utilization thresholds (Yellow â‰Ą0.70, Orange â‰Ą0.85, Red â‰Ą0.95) and scheduled interval logic; if all thresholds are below trigger, it fires E-02b to NO-OP-TERMINAL, terminating the pipeline cleanly in a few-token W1-only execution. Only when the trigger fires, or when mode is manual, does the pipeline proceed to W2.

Wave 2 and Wave 3 form the analysis backbone of the pipeline via two independent HC-23 parallel fan-outs dispatched each in a single response. In W2, four nodes execute simultaneously: N-GENERATOR builds the dependency DAG (graph_structure + dependency_graph + critical_path_set); N-CLASSIFIER identifies the conversation domain across five frames and emits domain gate signals; N-FILTER identifies always-retain and critical-path nodes; and N-FILE-CACHE performs SHA-256 delta comparison on any attached file references, emitting compact FileRef tokens for unchanged files to cap downstream token consumption. In W3, five nodes execute simultaneously using the W2 outputs: N-SCORER-TECHNICAL and N-SCORER-CREATIVE apply domain-specialized R(msg) scoring formulas, each activated only when the corresponding domain gate (E-09/E-10) is satisfied; N-SEMANTIC-CLUSTER segments the conversation into episodes and intra-cluster summarizes (inactive in MINIMAL); N-ANALYZER-TOPIC-SWITCH detects vocabulary-shift topic boundaries; and N-ANALYZER-CORRECTIONS tracks supersession chains to canonicalize latest-instruction-wins. Long-carry edges from N-INGEST (E-05b, E-16b), N-FILTER (E-17), and N-GENERATOR (E-08d) begin their cross-wave journeys here, carrying critical provenance and structural data to downstream aggregation and formatting nodes.

Wave 4 is the first major synthesis point. N-AGGREGATION collects 8 streams via AND-join: two required unconditional streams from N-INGEST and N-GENERATOR carrying session provenance and the dependency DAG; three unconditional analysis streams from N-FILTER, N-ANALYZER-TOPIC-SWITCH, and N-ANALYZER-CORRECTIONS; and three null-tolerant streams from N-SCORER-TECHNICAL, N-SCORER-CREATIVE, and N-SEMANTIC-CLUSTER (null-tolerant because these may not fire in MINIMAL mode or when domain gates evaluate false). The aggregation applies a 5-tier priority merge: tier 1 unconditionally protects always-retain and critical-path nodes (tool-call chains, dependency graph roots); tier 2 applies supersession canonicalization; tier 3 promotes the active topic thread; tier 4 applies semantic cluster bridging; and tier 5 blends domain scores by probability weight. The output is a single authoritative pruning_plan JSON consumed by the next spawn.

Waves 5 through 7 implement a self-correcting quality gate loop. N-REFINER (W5 spawn) applies the pruning_plan annotations to the full conversation corpus, substituting FileRef tokens for cached files and recording per-turn tier hints. N-VERIFIER (W6 inline) then runs a 13-SC safety battery against the refined context, checking for violations like pending tool-call eviction, critical-path disruption, or window-size overflow. If the battery passes, gate-open edge E-19 routes forward to N-TIER-MANAGER (W7 spawn). If the battery fails, back-edge E-20 fires to N-RECOVERY, which applies per-SC-ID override procedures and emits recovery_overrides. Back-edge E-21 then injects those overrides back into N-REFINER for a corrective second pass â€” after which N-VERIFIER runs again. The back-edge cap encodes quality-vs-latency policy: MINIMAL caps at 0 (no recovery, fastest path), STANDARD at 1 (one correction cycle), DEEP at 2 (two correction cycles for highest fidelity). Whether proceeding via E-19 (pass path) or E-19b (recovery-complete path), N-TIER-MANAGER executes Algorithm A: four eviction phases that detach completed tool chains, archive cold turns, compress warm turns to summaries, and emergency-evict non-critical turns by ascending R(msg) score when the budget still exceeds window_size.

Waves 8 and 9 handle formatting and persistence. N-FORMATTER assembles the final output: a YAML frontmatter block plus a 5-section body, incorporating the tier assignments from N-TIER-MANAGER (via both E-17b intermediate tier_state and E-22 final tier_state_final) and the always-retain exclusion list from N-FILTER (long-carry E-17 spanning 6 waves). If the pruning run was triggered automatically (auto_prune_triggered==true), forward-conditional edge E-23 fires to N-CONTINUITY-MARKER, which generates a human-readable pruning summary that can be injected as the first message in the pruned context, enabling transparent handoff without losing conversational coherence. If the run was manual or auto_prune was not triggered, E-24b routes directly from N-FORMATTER to N-PERSISTER, skipping the continuity step. N-PERSISTER (no-llm) writes the final artifact to disk and emits E-25 to SKILL_OUTPUT. The XOR join at N-PERSISTER (exactly one of E-24 or E-24b active per run) ensures N-PERSISTER always receives exactly one input signal, with null-at-persister treated as a halt condition.
